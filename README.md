# blog-static

Static blog for GeneXus, powered by [Eleventy](https://www.11ty.dev/). Content is authored in Google Docs and synced automatically to this repo.

## How it works

1. Authors write posts in a shared Google Drive folder, organized by category subfolders
2. A sync script pulls docs from Drive, converts them to Markdown, and writes them to `src/posts/`
3. Eleventy builds the static site from those Markdown files
4. The site is deployed to a CDN on every push

## Project structure

```
.secrets/                      Service account key file (git-ignored)
scripts/                       Sync pipeline (Drive → Markdown)
  auth.js                        Authenticates with Google APIs using the service account key
  config.js                      Central config: Drive folder ID and output paths
  list-docs.js                   Walks the Drive folder tree and returns all docs with metadata
  docs-to-markdown.js            Parses Google Docs structured JSON into Markdown
  sync.js                        Entry point — detects changes, runs the parser, writes .md files
src/                           Eleventy source files
  posts/                         Markdown blog posts (auto-generated by sync, do not edit by hand)
  _includes/                     Nunjucks layout templates (base.njk, post.njk)
  _data/                         Global data files available to all templates
  assets/css/                    Stylesheets
  index.njk                      Blog homepage — lists all posts
  feed.njk                       Atom RSS feed template
eleventy.config.js             Eleventy configuration: plugins, filters, input/output dirs
.gitignore                     Excludes .secrets/, node_modules/, _site/, sync-state.json
docs/
  checklist.md                   Implementation progress checklist
```

## Writing a post

1. Open the shared Drive folder
2. Navigate to a category subfolder (e.g., `Software Development`, `General Interest`) or create a new one
3. Create a Google Doc — the doc title becomes the post title
4. Optionally add a `tags: foo, bar, baz` line at the very top of the doc
5. To keep a post as draft, prefix the title with `[DRAFT]` (e.g., `[DRAFT] My New Post`)
6. To archive a post, prefix with `[ARCHIVED]`

### What gets extracted automatically

| Field    | Source                          |
|----------|---------------------------------|
| Title    | Google Doc title                |
| Slug     | Slugified doc title             |
| Author   | Drive file owner                |
| Date     | Drive file `createdTime`        |
| Category | Subfolder name                  |

## Local development

```bash
npm install        # Install dependencies
npm run sync       # Pull latest posts from Google Drive
npm start          # Start dev server at http://localhost:8080
npm run build      # Build the site to _site/
```

The sync script requires a Google service account key file in `.secrets/` (excluded from git via `.gitignore`). In CI, set the `GOOGLE_SERVICE_ACCOUNT_KEY` environment variable with the JSON contents instead.

## Configuration

- **Drive folder ID**: `scripts/config.js`
- **Eleventy settings**: `eleventy.config.js`
- **Site templates**: `src/_includes/`
